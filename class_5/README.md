# if文とswitch文の使い分け（class_5）

このフォルダには if文とswitch文の適切な使い分けを示すサンプルコードが含まれているでござる。

## ファイル
- `if_vs_switch.c` : 実践的な例で if文とswitch文の使い分けを説明するでござる。

## コンパイルと実行（macOS zsh, clang）
```bash
# コンパイル
clang -Wall -Wextra class_5/if_vs_switch.c -o class_5/if_vs_switch

# 実行
./class_5/if_vs_switch
```

## サンプル内容

1. 映画の年齢制限判定（if文の例）
   - 年齢による範囲判定でござる
   - 身分証の有無との複合条件でござる
   - 不正値のチェックも含むでござる

2. 色選択システム（switch文の例）
   - 文字コードによる単純な分岐でござる
   - 明確に定義された選択肢でござる
   - デフォルト処理の例も含むでござる

3. 季節の催し物案内（両方の実装比較）
   - 同じ処理を両方の方法で実装しているでござる
   - fall-through（意図的なbreak省略）の活用例でござる
   - 可読性の違いが分かる例でござる

## 使い分けの基準

### if文を使うべき場面
- 範囲を判定する場合（例：年齢制限）でござる
- 複数の条件を組み合わせる場合（AND や OR）でござる
- 不規則な条件分岐がある場合でござる
- 真偽値（0/1）による分岐でござる

### switch文を使うべき場面
- 単一の値による分岐の場合（例：色コード）でござる
- 値が限定されている場合でござる
- 連続した値をまとめて処理する場合でござる
- コードの見通しを良くしたい場合でござる

## 注意点
1. switch文ではbreak忘れに注意するでござる
2. if文では条件の順序に注意するでござる（範囲判定の順序など）
3. 可読性を重視して選択するでござる
4. 保守性も考慮するでござる（将来の変更のしやすさ）
